Cross-Sell Suggestion Agent (CSSA)

Software Project Management — Final Presentation

Awaiz Ali Khan (22I-2509) • Zain ul Abideen (22I-2738) • Kamran Ali (22I-2589)

SE4002 — Section SE-D | Instructor: Ma'am Behjat Zubair | Nov 2025

Problem Statement

  • Increase Average Order Value (AOV) via relevant cross-sell recommendations
  • Requirements: real-time data, intelligent matching, persistence, scalability
  • Solution: Supervisor–Worker (Registry) architecture with an autonomous agent

Project Objectives

  1. Implement CSSA with real external data (Fake Store API)
  2. Dual-memory: STM (in-memory) + LTM (SQLite)
  3. REST API contract, logging, health checks, and Docker deployment

System Architecture

Supervisor → HTTP → CSSA Agent (Flask)

Supervisor
  ↓
CSSA Agent
 ├─ API Layer
 ├─ Recommendation Engine
 ├─ Product Database (products.json)
 └─ Memory: STM + LTM (SQLite)

Technology Stack

  • Python 3.11, Flask 3.0
  • SQLite for LTM, JSON cache for product catalog
  • jsonschema for validation, pytest for tests, Docker for deployment

Memory Strategy

Short-Term Memory (STM)

In-memory per-session buffer, max 100 interactions, fast access for context.

Long-Term Memory (LTM)

SQLite `cssa_memory.db`, durable interactions, queryable by Supervisor.

API Contract (Sample)

POST /api/recommend

{ "session_id": "s1", "customer_products": [1,2,3], "limit": 5 }

Returns: recommendations array with confidence scores and reason.

Real Data Integration

  • Fetch from Fake Store API via `setup.py`
  • Cache results to `products.json` for portability
  • Fallback: cached JSON → hardcoded demo data (graceful degradation)

Demo Walkthrough

  1. Run: python setup.py (fetch products)
  2. Start agent: python cssa_agent.py
  3. UI: visit http://127.0.0.1:5000 and request recommendations
  4. Show Swagger and `/api/memory/{session_id}`

Key Metrics

  • Recommendation latency (p95): 45ms
  • Search latency (p95): 32ms
  • Test coverage (critical): 87%
  • Integration tests: 7/7 PASS

Challenges & Solutions

  1. 415 JSON errors → Added robust parsing fallback
  2. Swagger 404 → Served `/openapi.json` endpoint
  3. External API fragility → Caching + fallback
